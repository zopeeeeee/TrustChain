       ```
       RUN apt-get update && \
           apt-get install -y --no-install-recommends gcc libpq-dev ffmpeg && \
           rm -rf /var/lib/apt/lists/*
       ```

    2. Update docker-compose.yml:
       - Add a named volume `upload_data` to the volumes section
       - Mount it in the backend service: `upload_data:/data`
       - This gives the backend a persistent /data directory for uploaded files and extracted frames/audio
       - Keep existing ./backend:/app volume mount for hot-reload (it takes precedence for /app, /data is separate)

    3. Create Alembic migration backend/alembic/versions/002_add_error_message.py:
       - Adds `error_message` column (String(500), nullable=True) to uploads table
       - Down migration drops the column
       - Use `op.add_column("uploads", sa.Column("error_message", sa.String(500), nullable=True))`
       - Revision depends on 001_initial_schema (get the revision ID from 001 file)

    4. Update frontend/vite.config.ts to add API proxy:
       - In the server config, add proxy rule:
         ```typescript
         server: {
           proxy: {
             "/api": {
               target: "http://backend:8000",
               changeOrigin: true,
             },
           },
         }
         ```
       - Note: Inside Docker network, frontend container can reach backend via service name "backend". But the Vite dev server runs inside the frontend container, serving to the browser. The browser makes requests to localhost:5173/api which Vite proxies to backend:8000. However, "backend" hostname only resolves inside Docker network. Since the browser hits localhost:5173, and Vite server is inside the container, the proxy target should be "http://backend:8000" (container-to-container). This works because the Vite dev server (inside frontend container) proxies to the backend container.
       - Also update the API_BASE in frontend/src/lib/api.ts to use "/api" (relative) instead of absolute URL, since the Vite proxy handles routing.

    5. Rebuild and start Docker stack:
       - `docker compose build --no-cache`
       - `docker compose up -d`
       - Wait for services to be healthy
       - Verify FFmpeg is available: `docker compose exec backend ffmpeg -version`
       - Verify migration ran: `docker compose exec backend alembic current`
       - Run backend tests inside container: `docker compose exec backend pytest tests/ -x -q`
