  <action>
    1. Create `contracts/.gitkeep` (empty directory placeholder for Phase 5)

    2. Create `.env` at project root:
       ```
       POSTGRES_USER=trustchain
       POSTGRES_PASSWORD=trustchain
       POSTGRES_DB=trustchain_db
       DATABASE_URL=postgresql+asyncpg://trustchain:trustchain@db:5432/trustchain_db
       DEBUG=true
       ```

    3. Create `backend/Dockerfile`:
       - FROM python:3.11-slim
       - WORKDIR /app
       - Install system deps: apt-get update && apt-get install -y --no-install-recommends gcc libpq-dev && rm -rf /var/lib/apt/lists/*
       - COPY requirements.txt and pip install --no-cache-dir -r requirements.txt
       - COPY . .
       - EXPOSE 8000
       - CMD: alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000
       Note: CMD runs migrations THEN starts server (per research anti-pattern advice: run migrations via CLI, not in lifespan)

    4. Create `frontend/Dockerfile`:
       - FROM node:20-alpine
       - WORKDIR /app
       - COPY package.json package-lock.json* ./
       - RUN npm install
       - COPY . .
       - EXPOSE 5173
       - CMD ["npm", "run", "dev", "--", "--host", "0.0.0.0"]

    5. Create `docker-compose.yml` with 3 services per research doc:
       - db: postgres:16-alpine, env from .env (POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB), port 5432:5432, pgdata volume, healthcheck with pg_isready
       - backend: build context ./backend, port 8000:8000, volume mount ./backend:/app, env DATABASE_URL and DEBUG from .env, depends_on db with condition: service_healthy, command: bash -c "alembic upgrade head && uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload"
       - frontend: build context ./frontend, port 5173:5173, volumes ./frontend:/app and /app/node_modules (anonymous volume to prevent overwrite), depends_on backend, command: npm run dev -- --host 0.0.0.0
       - volumes: pgdata

    6. Create `.dockerignore`, `backend/.dockerignore`, `frontend/.dockerignore`:
       - backend: __pycache__, *.pyc, .venv, venv, .env, .git
       - frontend: node_modules, dist, .git
       - root: .git, __pycache__, node_modules

    7. Generate initial Alembic migration:
       Before docker compose up, we need an initial migration. Since we cannot run alembic without a DB, create the initial migration file manually:
       - Create `backend/alembic/versions/001_initial_schema.py` as a manual migration:
         - upgrade(): op.create_table("uploads", ...) with all columns from Upload model, op.create_table("blockchain_records", ...) with all columns from BlockchainRecord model including ForeignKey
         - downgrade(): op.drop_table("blockchain_records"), op.drop_table("uploads")
       - Use proper Alembic revision format with revision ID (use a short hex string like "0001abcd0001")
  </action>

  </verify>
  <done>docker-compose.yml validates, Dockerfiles exist for backend and frontend, .env has DB credentials, initial Alembic migration is ready</done>

    7. Run backend tests inside container: `docker compose exec backend pytest tests/ -x -q`

    - Rebuild: `docker compose up -d --build <service>`

    Leave containers running for the checkpoint task.
